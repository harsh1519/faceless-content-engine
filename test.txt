# PROJECT: Faceless Content Automation Engine (V1 - Free Stack)

## OVERVIEW
Build a web application that automates faceless short-form video content creation, 
distribution, and monetization tracking. This is a "mission control" dashboard 
similar to a social media management tool combined with an ads manager.

The system manages multiple channels across platforms (Instagram, YouTube, TikTok), 
moves content through a production pipeline, and tracks monetization via affiliate/CPA offers.

## TECH STACK (ALL FREE TIER - DO NOT DEVIATE)
- Frontend: Next.js 14 (App Router) + TypeScript + Tailwind CSS + shadcn/ui
- State/Data fetching: TanStack Query (React Query)
- Backend + DB + Auth: Supabase (Postgres, Row Level Security, Edge Functions)
- LLM for scripts: Google Gemini API (free tier) via @google/generative-ai
- TTS: ElevenLabs API (free tier)
- B-roll: Pexels API (free)
- Video rendering: Local FFmpeg via a Node.js script (run on user's machine)
- Charts: Recharts
- Icons: lucide-react

   - channel_id (uuid, pk, default gen_random_uuid())

   - platform (text: 'instagram' | 'youtube' | 'tiktok')
   - niche_type (text)
   - target_demographics (text)

   - trend_id (uuid, pk)
   - source (text)
   - keyword (text)
   - hook_text (text)
   - velocity_score (numeric)
   - predicted_epc (numeric)  -- earnings per click prediction
   - status (text: 'new' | 'approved' | 'rejected', default 'new')
   - created_at (timestamptz, default now())

   - offer_id (uuid, pk)
   - name (text)
   - offer_type (text: 'cpa_lead' | 'cpa_sale' | 'affiliate')
   - payout (numeric)
   - vertical (text)
   - affiliate_url (text)
   - cloaked_url (text)
   - status (text: 'active' | 'paused', default 'active')
   - created_at (timestamptz, default now())

   TABLE content_objects:
   - video_id (uuid, pk)
   - channel_id (uuid, fk -> channels)
   - trend_id (uuid, fk -> trends, nullable)
   - offer_id (uuid, fk -> offers, nullable)
   - script (text)
   - audio_path (text, nullable)
   - render_path (text, nullable)
   - status (text: 'trend_queue' | 'script_review' | 'rendering' | 'ready_approve' | 'published' | 'failed', default 'script_review')
   - thumbnail_url (text, nullable)
   - published_at (timestamptz, nullable)
   - created_at (timestamptz, default now())

   TABLE leads:
   - lead_id (uuid, pk)
   - source_video_id (uuid, fk -> content_objects, nullable)
   - contact_channel (text: 'email' | 'telegram' | 'sms')
   - contact_value (text)
   - consent_status (boolean, default false)
   - intent_tags (jsonb, default '[]')
   - created_at (timestamptz, default now())

   TABLE conversions:
   - conversion_id (uuid, pk)
   - video_id (uuid, fk -> content_objects, nullable)
   - offer_id (uuid, fk -> offers)
   - click_count (int, default 0)
   - conversion_count (int, default 0)
   - revenue (numeric, default 0)
   - occurred_at (timestamptz, default now())

6. Add a seed file (supabase/seed.sql) with 3 sample channels (different niches),

## PHASE 8: LOCAL VIDEO RENDERER (Separate Node Script)

Create a standalone Node.js script in /renderer/render.js (NOT in the Next app) that:
1. Polls Supabase for content_objects with status 'rendering'.
2. For each: downloads the audio + broll clips, uses FFmpeg (via fluent-ffmpeg) to:
   - Concatenate broll clips (each clip max 2.5 seconds for retention)
   - Overlay the audio track
   - Add auto-generated captions (use the script text, simple subtitle overlay)
   - Output 1080x1920 mp4
3. Uploads the result to Supabase Storage, updates render_path + status to 'ready_approve'.
4. Include clear README instructions for installing FFmpeg locally and running this script.
Make it robust with try/catch and set status to 'failed' on error.
